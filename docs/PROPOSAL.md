# VR_Tutorial 기획서

## 1. 개요
방문자가 본 VR 게임을 시작하기 전 **5~10분 안에 필수 인터랙션을 익히도록** 하는
온보딩 씬. Unity XR Interaction Toolkit 기반으로 제작하며, Meta Quest 2/3 환경을 1차 타겟으로 한다.

## 2. 배경 / 문제정의
- 현장 방문자 다수가 VR 첫 경험이며 컨트롤러·이동 방식에 어색함을 느낌
- 본 게임에서 곧바로 시작하면 멀미·조작 미숙으로 이탈이 발생
- 운영자가 1:1로 설명하면 대기열이 발생하고 처리량이 떨어짐

본 튜토리얼이 상기 세 가지를 줄이는 것을 목표로 한다.

## 3. 타겟 사용자
- VR 헤드셋 첫 경험자 (운영 현장 일반 방문자)
- 한국어 사용자 (UI/내레이션 한국어 우선)
- 평균 세션 길이: 5~10분
- 동시 접속: 단일 헤드셋 단일 사용자 기준

## 4. 학습 목표
세션 종료 시점에 사용자가 자력으로 다음을 수행할 수 있어야 한다.

1. 머리 회전·둘러보기로 환경 인지
2. 양손 컨트롤러의 트리거·그립 버튼 인지 및 사용
3. 오브젝트 집기·놓기 (Direct Grab + Ray Grab 모두)
4. 텔레포트로 지정 위치로 이동
5. 월드 공간 UI 버튼 클릭
6. 일시정지·튜토리얼 종료 메뉴 호출

## 5. 핵심 기능
- **단계별 가이드 패널**: 현재 미션 텍스트 + 컨트롤러 부위 시각 하이라이트
- **미션 성공 판정**: Trigger Volume + 이벤트 콜백으로 자동 판정
- **스킵·재시작 옵션**: 운영자/숙련자용 우회 경로
- **진행 상황 저장**: PlayerPrefs 로 마지막 단계 기록
- **본 게임 씬 전환**: 완료 후 자연스럽게 메인 씬으로 이동

## 6. 기술 스택
| 영역 | 선정 |
|---|---|
| 엔진 | Unity 2022.3 LTS |
| XR | XR Interaction Toolkit 2.5+ |
| Plugin | OpenXR Plugin (Meta Quest Feature Group) |
| UI | TextMeshPro, World-space Canvas |
| 입력 | Unity Input System (XRI Default Input Actions 베이스) |
| 오디오 | Unity Audio + 한국어 내레이션 mp3 |
| (선택) | Unity Localization (다국어 확장 시) |

## 7. 비기능 요구사항
- Quest 2 기준 90 FPS 안정 유지 (드로우 콜 ≤ 50, 트라이앵글 ≤ 100k)
- 첫 씬 로딩 5초 이내
- 헤드셋 미장착 시 자동 일시정지

## 8. 마일스톤
| 주차 | 산출물 |
|---|---|
| W1 | 씬 골격, XR Rig 셋업, Bootstrap → MainMenu → Tutorial 흐름 |
| W2 | 미션 1~3 (둘러보기 · 컨트롤러 인식 · 직접 그랩) |
| W3 | 미션 4~6 (레이 그랩 · 텔레포트 · UI 클릭) |
| W4 | 미션 7~8 (통합 챌린지 · 종료) + 사운드/내레이션 |
| W5 | QA, Quest 빌드 검증, 운영자 매뉴얼 |

## 9. 성공 지표
- 튜토리얼 완료율 ≥ 80%
- 평균 소요시간 7~10분 내
- 본 게임 진입 후 첫 미션 실패율 (튜토리얼 시행 전 대비) **유의미한 감소**
- 운영자 1:1 설명 요청 빈도 감소

## 10. 향후 확장 (Out of Scope, 차후)
- 다국어 (영어, 일본어)
- 멀티 사용자 동시 튜토리얼
- 손 트래킹 (Hand Tracking) 모드
